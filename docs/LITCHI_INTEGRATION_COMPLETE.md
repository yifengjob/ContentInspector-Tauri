#### 2.1 新增辅助函数

```rust
/// 使用 litchi 库提取 Office 文档文本
fn extract_with_litchi(path: &str) -> Result<String, String> {
    // 尝试作为 Document 打开（支持 .doc/.docx）
    match litchi::Document::open(path) {
        Ok(doc) => {
            doc.text().map_err(|e| format!("litchi 提取文本失败: {}", e))
        }
        Err(e) => {
            // 如果不是 Word 文档，尝试作为 Presentation 打开（支持 .ppt/.pptx）
            match litchi::Presentation::open(path) {
                Ok(pres) => {
                    pres.text().map_err(|e| format!("litchi 提取 PPT 文本失败: {}", e))
                }
                Err(_) => {
                    Err(format!("litchi 无法识别文件格式: {}", e))
                }
            }
        }
    }
}
```

#### 2.2 改造的函数

1. **`read_doc_file()`** - 读取DOC文件
   - 方案1: 使用litchi提取（高质量）
   - 方案2: 降级到二进制扫描（兼容性）

2. **`read_ppt_file()`** - 读取PPT文件
   - 方案1: 使用litchi提取（高质量）
   - 方案2: 降级到二进制扫描（兼容性）

3. **`stream_extract_doc()`** - 流式提取DOC
   - 方案1: litchi提取后按行分块回调
   - 方案2: 降级到分块二进制扫描

4. **`stream_extract_ppt()`** - 流式提取PPT
   - 方案1: litchi提取后按行分块回调
   - 方案2: 降级到分块二进制扫描

### 3. 降级机制设计

```
┌─────────────────────┐
│  调用 read_doc_file  │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 尝试 litchi 解析     │
└──────────┬──────────┘
           │
     ┌─────┴─────┐
     │           │
  成功✅      失败❌
     │           │
     ▼           ▼
 返回文本   记录警告日志
             │
             ▼
      降级到二进制扫描
             │
             ▼
      返回简化文本
```

**优势**:
- ✅ 大部分标准DOC文件获得高质量文本提取
- ✅ WPS生成的非标准文件也能通过降级机制处理
- ✅ 不会因解析失败而崩溃
- ✅ 详细的日志记录便于调试

## ⚠️ 已知限制

### 1. WPS兼容性问题

**问题**: WPS Office生成的某些小文件（~60KB）缺少标准的OLE组件（如0Table），导致litchi无法解析。

**影响**: 
- 约30%的测试文件无法使用litchi解析
- 这些文件会回退到低质量的二进制扫描

**解决方案**:
- 保持降级机制
- 建议用户转换为标准DOCX格式

### 2. 内存占用

**问题**: litchi一次性加载整个文件，对于超大文件（>500MB）可能占用较多内存。

**解决方案**:
- 对大文件（>100MB）自动切换到降级模式
- 或者在stream_extract_*函数中实现真正的流式处理

### 3. API稳定性

**问题**: litchi仍处于早期开发阶段（v0.0.1），API可能变化。

**解决方案**:
- 锁定版本号 `litchi = "0.0.1"`
- 定期关注更新
- 准备备用方案
